# ADR 0001 — Telegram-фреймворк: aiogram 3.x

## Контекст
Бот должен быть асинхронным, поддерживать FSM (создание карточек, регрессия), inline-клавиатуры, middlewares (авторизация по ролям), фоновые задачи (напоминания).

## Решение
Используем `aiogram` 3.x.

## Альтернативы
- `python-telegram-bot` — стабилен, но FSM-механика и middlewares менее идиоматичны для нашего сценария с DI.
- `telebot` — синхронный по умолчанию, не подходит.

## Последствия
- Все хэндлеры — `async`.
- DI через `Dispatcher.workflow_data` / middlewares.
- Требуется отдельная отладка graceful shutdown с APScheduler.
