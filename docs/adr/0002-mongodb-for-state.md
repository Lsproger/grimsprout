# ADR 0002 — MongoDB для состояния бота

## Контекст
Нужно хранить: пользователей и роли, текущую сессию (выбранное растение), расписания напоминаний с jobstore для APScheduler, аудит-лог.

## Решение
MongoDB сразу (как в исходном ТЗ). Драйвер — `motor` (async).

## Альтернативы
- SQLite + миграция на Mongo позже — меньше зависимостей, но удваивает работу.
- Только YAML/файлы — не подходит для расписаний и аудита.

## Последствия
- В docker-compose добавляется сервис `mongo`.
- APScheduler использует `MongoDBJobStore`.
- Карточки растений в Mongo **не** дублируются — источник истины остаётся в `trava/*.md`.
