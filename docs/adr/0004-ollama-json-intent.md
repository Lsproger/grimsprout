# ADR 0004 — LLM возвращает JSON-интент (без tool calls)

## Контекст
Локальная LLM через Ollama должна извлечь из свободного текста структурированное намерение пользователя.

## Решение
На MVP — LLM возвращает строгий JSON по фиксированной схеме (`docs/spec/04-llm-contract.md`). Никаких tool/function-calls. Все мутации выполняет код бота.

## Альтернативы
- Function calling / tool use — добавляет требования к модели, усложняет MVP.
- Свободный текст с парсингом regex — нестабильно.

## Последствия
- Используем Ollama `format: "json"` для жёсткости вывода.
- При ошибке валидации — один retry + fallback.
- Архитектура готова к переходу на tool calls / MCP без переписывания core (см. ADR 0005).
