# ADR 0007 — Переход на нативный tool calling (Фаза 4.5)

**Дата:** 2026-05-28  
**Статус:** Принято  
**Заменяет:** ADR 0004

## Контекст

Архитектура Фазы 3 (JSON-интент через `format: "json"`) столкнулась с тремя системными проблемами:

1. **«Слепота» к растениям.** LLM не получала список карточек и изобретала ID («areca», «Areca palm», «areca_palm» вместо реального `areca_01`).

2. **Неявные намерения и размытые границы.** Одна модель одновременно маршрутизировала действия, отвечала на вопросы и возвращала строгий JSON — три задачи с противоречивыми требованиями к стилю вывода.

3. **Отсутствие батч-операций.** Архитектура JSON-интента не имела механизма «применить ко всем растениям».

Модели на базе Gemma 4 (в частности `gemma4:e4b`) поддерживают нативный tool calling стандарта OpenAI, что позволяет перейти без смены провайдера.

## Решение

С Фазы 4.5 GrimSprout использует **нативный tool calling** через `python-ollama` (`ollama.AsyncClient.chat(tools=...)`).

**Ключевые изменения:**

1. **Два агента вместо одного:**
   - *Classifier* — модель с tool calling, маршрутизирует намерение через инструменты.
   - *Assistant* — та же или другая модель, отвечает на свободные вопросы без инструментов.

2. **Repo summary injection.** Каждый запрос к Classifier включает вывод `build_repo_summary()` — компактную сводку всех карточек с ID, статусом и последними changelog-записями. Устраняет «слепоту».

3. **6 инструментов** покрывают все действия: `water`, `fertilize`, `repot` (поддерживают `["all"]`), `observe`, `get_plant_details` (read-only), `create_plant` (read-only, возвращает подсказку `/new`).

4. **`PendingMutation` / `AgentResult`** заменяют `Intent`. Мутирующие вызовы собираются в `pending_mutations`; если `confirm_commits=true` — показывается превью перед записью.

## Альтернативы

- **Оставить JSON-интент, добавить список растений в промпт** — частично решает «слепоту», но не устраняет проблему одного промпта на всё; retry-логика остаётся.
- **MCP (Model Context Protocol)** — перспективная цель (ADR 0005), но требует отдельного сервера; инструменты — более лёгкий путь к той же архитектуре.
- **Свободный текст + regex** — нестабильно, отклонено ещё в ADR 0004.

## Последствия

- `intent_parser.py`, `system_undertaker.md`, `intent_schema.json` — сохранены, но не используются.
- `format: "json"` и retry-on-invalid-JSON — удалены из пути выполнения.
- Требования к модели выросли: нужна поддержка tool calling. Все модели на базе `gemma4` на текущем инстансе Ollama поддерживают.
- Путь к MCP-экспорту (ADR 0005) стал короче: инструменты уже описывают API GrimSprout в виде JSON-схем.
