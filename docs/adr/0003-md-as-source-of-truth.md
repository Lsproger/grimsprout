# ADR 0003 — Markdown-карточки как источник истины

## Контекст
Растения уже учитываются в Git-репозитории `trava` в виде Markdown + YAML front matter + changelog. Эти файлы — основной артефакт пользователя.

## Решение
Карточки растений остаются источником истины. MongoDB хранит **только** бот-состояние (пользователи, сессии, расписания, аудит). Любые правки расписаний и метаданных, влияющих на растение, отражаются в `.md` через атомарную запись.

## Последствия
- При расхождении Mongo и `.md` побеждает `.md`.
- Сделать бота можно использовать без Mongo для чтения карточек (read-path не требует БД).
- Это создаёт естественную границу для будущего MCP-сервера (см. ADR 0005).
