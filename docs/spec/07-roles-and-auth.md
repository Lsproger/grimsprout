# 07 — Роли и авторизация

## 7.1. Роли
| Роль | Описание |
|---|---|
| `admin` | Всё. Управление пользователями, push, любые правки. |
| `publisher` | Все правки + `/push`. |
| `editor` | Правки карточек, расписания, локальные коммиты. Без push. |
| `viewer` | Только просмотр (`/plants`, чтение карточки). |

## 7.2. Матрица прав

| Действие | viewer | editor | publisher | admin |
|---|:-:|:-:|:-:|:-:|
| `/start`, `/help`, `/whoami` | ✅ | ✅ | ✅ | ✅ |
| `/plants` (список, выбор) | ✅ | ✅ | ✅ | ✅ |
| Прямые команды (`/water`, `/fertilize`, `/repot`) | ❌ | ✅ | ✅ | ✅ |
| Свободный текст → LLM → мутации | ❌ | ✅ | ✅ | ✅ |
| Приём фото, запись в changelog | ❌ | ✅ | ✅ | ✅ |
| `/new` | ❌ | ✅ | ✅ | ✅ |
| Расписания | ❌ | ✅ | ✅ | ✅ |
| `/push` | ❌ | ❌ | ✅ | ✅ |
| Управление пользователями | ❌ | ❌ | ❌ | ✅ |

## 7.3. Регистрация
- Стартовый `admin` задаётся в `config.yaml` (`telegram.bootstrap_admin_tg_id`); создаётся в коллекции `users` при первом запуске, если её там ещё нет.
- Другие пользователи добавляются только админом через `/add_user <tg_id> <role>`.
- При попытке доступа неизвестного — silent ответ + уведомление админу + запись в `audit_log`.

## 7.4. Реализация
- Middleware `AuthMiddleware` загружает `users.find_one({tg_id})` и кладёт `user` в data; неавторизованные обрабатываются отдельным fallback-хэндлером.
- Декоратор `requires_role(*roles)` на handler-уровне.
