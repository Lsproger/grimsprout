# 01 — Обзор

GrimSprout — Telegram-агент для автоматизации персонального репозитория учёта домашних растений `trava`. Бот принимает сообщения и фото от авторизованного владельца, парсит интент через локальную LLM (Ollama), редактирует Markdown-карточки растений и фиксирует изменения в Git.

Исходное техническое задание: см. [tz.md](../../tz.md). Документы в `docs/spec/` уточняют контракты и архитектуру, документы в `docs/adr/` фиксируют ключевые архитектурные решения.

## Принципы
- **Источник истины по растениям** — Markdown-файлы репозитория `trava` (YAML front matter + changelog). MongoDB не дублирует карточки.
- **Состояние бота** (пользователи, роли, сессии, расписания, аудит) — в MongoDB.
- **LLM возвращает строгий JSON-интент.** Никаких сторонних эффектов из LLM; все мутации выполняет код бота.
- **Git push — только вручную** (`/push`) пользователем с ролью `publisher` или `admin`. Автокоммит — после каждой правки.
- **Модульный core** (чистые операции с файлами/Git) отделён от адаптеров Telegram/LLM/фото — задел под будущий MCP-сервер.
