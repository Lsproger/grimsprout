# Команда `/new` — FSM-создание новой карточки растения

**Фаза**: 4
**Статус**: done
**Приоритет**: high
**Зависимости**: нет

## Описание

Пользователь хочет завести новое растение в склепе прямо из Telegram, не создавая markdown-файл вручную. FSM проводит через 7 шагов: название, научное название, сорт, дата покупки, место покупки, возрастная группа, подтверждение.

## Критерии готовности

- [x] `/new` запускает FSM, очищая предыдущее состояние
- [x] Каждый шаг принимает `/skip` для пропуска необязательных полей
- [x] Возрастная группа выбирается через inline keyboard
- [x] Предпросмотр карточки перед подтверждением
- [x] `slugify(common_name)` + `next_slug()` генерирует уникальный `plant_id`
- [x] `md_parser.write()` создаёт `.md` файл с YAML front matter и Changelog
- [x] `git add` + `git commit` фиксирует новую карточку
- [x] Audit-запись с `action=create_plant`
- [x] `DirtyRepoError` и `GrimSproutError` обрабатываются с user-friendly ответом

## Заметки

- Реализовано в `src/grimsprout/bot/handlers/new_card.py`
- Использует `core/ids.py`: `slugify` + `next_slug` для генерации ID
- `NewCardFSM` определён в `bot/states.py` (был готов заранее)
