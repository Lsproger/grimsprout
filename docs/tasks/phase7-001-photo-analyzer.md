# PhotoAnalyzer: LLM Vision

**Фаза**: 7
**Статус**: backlog
**Приоритет**: low
**Зависимости**: phase3-001-ollama-client

## Описание

Реализовать LLM-based анализ фотографий растений: описание состояния, определение заболеваний/вредителей, автоматическое добавление описания в changelog при загрузке фото.

**Зачем**: Пользователь загружает фото — бот автоматически описывает что видит (состояние листьев, признаки болезней) и записывает это в changelog, снижая необходимость ручного описания.

**Кто**: editor+ (при загрузке фото)

**Что**: При получении фото бот отправляет изображение в LLM с vision-поддержкой, получает структурированное описание, предлагает добавить его в changelog.

## Критерии готовности

- [ ] `PhotoAnalyzer` Protocol реализован (stub уже в `services/photo_analyzer.py`)
- [ ] Интеграция с Ollama vision-моделью (например `llava` или `gemma3`+vision)
- [ ] Возврат: `{"health_signals": [...], "description": "...", "diseases": [...]}`
- [ ] При загрузке фото: анализ → предпросмотр описания → подтверждение → changelog entry
- [ ] Флаг включения в конфиге (`llm.photo_analyzer_enabled: bool = False`)
- [ ] Graceful fallback: если модель недоступна — запись без описания (как сейчас)

## Заметки

- `StubAnalyzer` уже в `src/grimsprout/services/photo_analyzer.py` — возвращает `{"available": False}`
- Использует существующий photo handler (`bot/handlers/photo.py`) как точку входа
- Ollama vision требует модель с multimodal поддержкой — проверить доступность на сервере
- Не входит в MVP
